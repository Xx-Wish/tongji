# 日志系统使用说明

## 📋 概述

程序现在有完整的日志系统，可以帮助您快速定位和解决问题。

## 📂 日志文件位置

日志文件存储在项目的 `logs` 目录中：
```
政策-学术文献交叉印证加权分析小工具/
└── logs/
    └── app_20250514.log  ← 每日一个日志文件
```

## 🔍 如何查找日志

### 方式1：手动查看
1. 打开项目文件夹
2. 进入 `logs` 文件夹
3. 找到对应日期的日志文件，用文本编辑器打开

### 方式2：告诉助手
当您遇到问题时，请直接对我说：
> "我遇到了问题，请查看日志帮我分析"

我会自动读取日志文件帮您诊断问题。

## 📊 日志级别说明

| 级别 | 颜色 | 说明 |
|------|------|------|
| DEBUG | 灰色 | 详细信息，用于调试 |
| INFO | 蓝色 | 一般信息，程序正常运行 |
| WARNING | 黄色 | 警告信息，需要注意 |
| ERROR | 红色 | 错误信息，出现问题 |
| CRITICAL | 深红 | 严重错误，程序可能崩溃 |

## 🐛 常见问题与日志

### 问题1：PDF解析失败
查看日志中是否有：
```
FileParseThread - 文件解析失败: ...
```

### 问题2：大模型调用失败
查看日志中是否有：
```
LLMClient - OpenRouter API调用失败: ...
```

### 问题3：数据提取为空
查看日志中是否有：
```
PaperDataExtractThread - 数据提取完成，提取到 0 条数据
```

## 💡 提示

- 日志按天分割，每天一个新文件
- 旧日志不会自动删除，可以查看历史问题
- 日志会记录所有关键操作，包括文件解析、大模型调用、数据提取等
- 如果遇到问题，请先查看最新的日志文件
