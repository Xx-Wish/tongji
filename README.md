# 大创项目背景数据生成工具

> 基于 PyQt5 的 Windows 桌面工具，帮助大学生创新创业项目快速生成标准化的背景数据。

---

## 项目简介

本工具专为**大学生创新创业项目**（大创、竞赛、立项申报）设计，解决以下痛点：

- 写申报书时缺乏**权威背景数据**支撑
- 参考文献多、乱、杂，不知道哪些数据和项目高度相关
- 手动整理数据耗时耗力，且来源难以追溯
- 缺乏系统性的**数据审核与评分**机制

本工具通过接入大模型 API，自动从导入的参考文献/政策文档中提取**6 类量化数据**，并经过四维度审核评分，最终导出为 Excel / Word 报告。

---

## 功能特性

| 功能 | 说明 |
|------|------|
| 🤖 多模型支持 | 支持豆包、通义千问、OpenRouter（Qwen）等主流大模型 API |
| 🔐 API Key 加密存储 | 所有密钥使用 `cryptography` 库加密后本地保存，绝不上传服务器 |
| 📄 多格式文件导入 | 支持导入项目申报书、参考文献（PDF / Word / TXT） |
| 🔍 智能文档识别 | 自动识别导入文档类型（申报书 or 参考文献），分别处理 |
| 📊 6 类数据提取 | 领域现状、用户分析、政策支撑、痛点缺口、竞争力对比、创新点支撑 |
| ✅ 四维度审核评分 | 时效性、交叉印证度、项目相关性、内容价值，按总分排序 |
| 📎 一键导出 | 支持导出为 Excel（`.xlsx`）和 Word（`.docx`）格式报告 |
| 🖥️ 单文件 EXE | 使用 `PyInstaller` 打包，免安装运行 |
| 📝 操作日志 | 内置日志系统，记录所有关键操作，便于排查问题 |

## 使用流程

### 第一步：配置 API

1. 打开程序，进入 **「API 配置 & 规则」** 标签页
2. 选择使用的模型（豆包 / 通义千问 / OpenRouter）
3. 填入对应模型的 **API Key**（将加密保存在本地）
4. 点击 **「保存配置」**
5. （可选）若需要解析 PDF 文件，还需配置百度 OCR 的 API Key 和 Secret Key

### 第二步：导入文件

进入 **「生成数据」** 标签页：

1. 填入**项目标题**
2. 点击 **「导入申报书」**，选择项目申报书文件（PDF / Word / TXT）
3. 点击 **「导入参考文献」**，选择参考文献文件（支持多个文件分批导入）
4. 导入后程序会自动解析文件内容

### 第三步：提取数据

导入完成后，程序会提示是否使用大模型提取结构化数据：

- 点击 **「是」**：程序将调用大模型，从文档中提取 6 类量化数据
- 点击 **「否」**：仅解析文档文字，后续生成时再调用大模型分析

### 第四步：生成数据

1. 点击 **「开始生成数据」**
2. 程序将：
   - 深度理解项目申报书（提取主题、研究方向、创新点等）
   - 对参考文献进行四维度审核评分
   - 按总分排序，筛选最相关的数据
3. 生成完成后，结果将展示在 **「结果展示 & 导出」** 标签页

### 第五步：导出结果

在结果页面，点击：

- **「导出 Excel」**：导出为 `.xlsx` 文件，便于进一步编辑
- **「导出 Word」**：导出为 `.docx` 文件，可直接用于申报书附录

---

## 四维度审核评分规则

每条数据都会经过以下四个维度打分，按**总分降序**排列：

| 维度 | 权重 | 说明 |
|------|------|------|
| 时效性 | 25 分 | 数据发布时间越近，得分越高 |
| 交叉印证度 | 15 分 | 是否有多个来源共同印证该数据 |
| 项目相关性 | 50 分 | 数据与项目主题、创新点的匹配程度（**核心维度**） |
| 内容价值 | 10 分 | 数据本身的权威性和引用价值 |

> ✅ 用户直接导入的文档数据（非网络搜索结果）**免审核**，直接保留。

## 6 类输出数据说明

程序最终会输出以下 6 类数据（根据导入内容自动决定输出哪些）：

1. **项目背景领域现状量化数据** — 行业规模、增速、研究热度等
2. **目标用户详细分析数据** — 用户画像、行为特征、需求分布等
3. **相关政策支撑数据** — 国家政策、地方支持、资金扶持等
4. **痛点与行业缺口量化数据** — 现存问题、服务盲区、市场空白等
5. **项目竞争力对比数据** — 同类产品对比、竞争格局分析等
6. **项目创新点支撑数据** — 研究空白、技术创新佐证等

---

## 强制规则（程序写死，不可突破）

1. **所有输出数据 100% 来自用户导入的文献/政策**，大模型仅负责统计分析，不编造任何无来源数据
2. **重点统计与项目高度相关的内容**，避免输出无关数据
3. **所有数据必须标注来源**，100% 可追溯
4. **项目申报书仅用于理解项目思路**，不作为输出数据的来源
5. **所有用户数据、API Key 仅本地存储**，绝不上传任何服务器

## 常见问题

**Q：程序支持 macOS / Linux 吗？**
A：理论上 PyQt5 跨平台，但本工具主要针对 Windows 优化，部分路径和字体设置可能需要调整。

**Q：API Key 会泄露吗？**
A：不会。所有 API Key 使用 `cryptography` 库的 Fernet 加密后保存在本地，程序不会将任何用户数据上传至服务器。

**Q：百度 OCR 是必须的吗？**
A：不是。只有需要解析**扫描版 PDF**（图片格式，无法直接提取文字）时才需要配置百度 OCR。

**Q：生成的数据可以直接用在申报书里吗？**
A：工具提取的数据仅供参考和支撑，请务必**人工核实每条数据的来源和准确性**后再使用。
